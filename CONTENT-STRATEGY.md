# 📚 内容策略 — 知识库与工具集的持续进化

> **核心理念：AI 发展太快，我们的内容必须跟上。新闻只是点缀，知识库和工具集才是核心价值。**

---

## 🎯 优先级排序（从高到低）

### P0 — 知识库持续进化（最高优先级）

知识库是网站的灵魂。188 篇文章不是终点，而是起点。

#### 1. 发现新方向 → 新增文章
每次心跳必须主动搜索 AI 前沿，发现值得写入知识库的新技术/新概念。

**搜索来源（不要局限在某一个渠道）：**
- arXiv cs.AI / cs.LG / cs.CL 最新论文
- OpenAI Blog、Anthropic News、Google DeepMind Blog
- HuggingFace 新模型 / 新 Space
- GitHub trending AI 项目
- Simon Willison、One Useful Thing、TechCrunch AI

**判断标准：**
- 这个技术/概念是否重要到值得写一篇文章教别人？
- 是否代表了 AI 发展的新方向？
- 用户学习 AI 时是否会需要这个知识点？

#### 2. 更新旧文档 → 保持时效性
**老文档不能发霉！** AI 领域的知识 3 个月可能就过时了。

**更新频率：**
- 热门主题（LLM、Agent、RAG、Diffusion 等）→ 每月检查
- 成熟主题（数学基础、经典 ML 算法）→ 每季度检查

**触发条件：**
- 该领域有重大突破（如 GPT-5 发布、新训练范式）
- 工具/框架有大版本更新（如 PyTorch 3.0、LangChain 重构）
- 已有文章的事实描述过时（版本号、API 变更、最佳实践变化）

**更新方式：**
- 不删除原有内容，在相关章节末尾追加"📅 更新于 YYYY-MM-DD：..."
- 新增"最新动态"章节记录最新进展
- 修正事实错误（如版本号、已废弃的 API）

#### 3. 质量标准
- 每篇至少 7 章结构（概念 → 原理 → 实战 → 代码 → 对比 → 注意事项 → 扩展阅读）
- 不能是几句话的摘要
- 代码示例必须可运行
- Mermaid 图表必须正确渲染

---

### P1 — 工具集紧跟时代（高优先级）

AI 工具每天冒出来，工具集必须是活的。

#### 1. 发现新工具
- GitHub Trending AI 项目（周增长 >1000 stars）
- **🔴 GitHub Topics 扫描（新增核心策略）**：
  - 维护本地 `data/ai-topics.json` AI 相关 topics 库
  - 每次巡检按 topic 搜索高星项目（`topic:xxx stars:>minStars`）
  - 对比 tools.ts 已有项目，发现遗漏立即收录
  - 发现新 topic → 更新本地 topic 库
- HuggingFace 新模型 / 新 Space
- Product Hunt AI 分类
- 各大公司最新发布（OpenAI、Anthropic、Google、Meta、NVIDIA）

#### 2. 更新老工具信息
- 工具的版本、定价、功能会变
- 每季度至少检查一次主流工具的最新信息
- 已停止维护的工具要标注或移除

#### 3. 质量标准
- 每个工具至少包含：pros[]、cons[]、useCase、learnMore
- 不能只有一句话 description

---

### P2 — 技术博客深度思考（每日更新）

- **每天至少更新/新增 1 篇博客**
- 不能只是改错别字，要有实质内容
- **格式要求（与知识库统一）：**
  - 使用 `ArticleSection[]` 结构化格式（不是 markdown 字符串）
  - 每篇至少 2 个 Mermaid 图（架构图/流程图/对比图/时间线）
  - 每篇至少 2 个 Python 代码块（可运行的示例代码）
  - 至少 1 个对比表格
  - 章节按 `body/code/mermaid/table/list/tip` 拆分
- 不能简单翻译搬运，要有原创视角
- 选题方向：最新论文深度解读、技术趋势分析、实战指南

---

### P3 — 首页新闻（最低优先级）

**新闻只是为了让首页看起来是活的，不是核心价值。**

- 每天更新 2-5 条即可
- 不要花太多时间
- 来源多样化，不依赖单一 API

---

## 🔧 数据收集策略

**绝对不要说"API 配额用完无法获取"这种话。** 方法总比困难多。

| 渠道 | 适用场景 | 备注 |
|------|---------|------|
| `web_fetch` 直接抓取 | 所有网站 | 最可靠，不依赖 API key |
| 浏览器 | 需要 JS 渲染的页面 | 备用，有时会超时 |
| GitHub API | trending 项目 | 需 token，否则限频 |
| arXiv RSS | 最新论文 | https://arxiv.org/rss/cs.AI |

**铁律：如果某个渠道不可用，立即换另一个。**
