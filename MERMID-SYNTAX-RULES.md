# Mermaid 图表语法铁律

## ✅ 安全的语法（mermaid v11.14.0）

### 流程图 - 用 `graph` 不是 `flowchart`
```
graph LR
    A[节点A] --> B[节点B]
    B --> C{判断?}
    C -->|是| D[结果1]
    C -->|否| E[结果2]
```

### 单节点样式
```
style A fill:#1e3a5f,stroke:#2563EB
```

## ❌ 禁止的语法（会导致渲染失败且不报错）

| 禁用语法 | 原因 | 替代方案 |
|---------|------|---------|
| `sequenceDiagram` | v11 不兼容 | 用 `graph LR/TD` |
| `class B,C orange`（逗号分隔多节点） | 语法解析失败 | 每个节点单独写 style |
| `F -.text.-. A`（虚线箭头） | 偶尔失败 | 用 `F --> A` 或 `F -.-> A` |
| `classDef` + `class` | 偶尔失败 | 直接在每个节点写 `style` |
| 节点名包含特殊字符 | 解析错误 | 只用字母数字 |
| 节点文本含引号/括号 | 解析错误 | 用「」替代引号 |

## 验证规则
写完后必须确认图表在浏览器中能正常显示。
