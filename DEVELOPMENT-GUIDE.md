> 最后更新：2026-03-29  
> 版本：v1.0

---

## 📋 核心原则

### 1. 质量优先于数量

- ❌ 不追求题目数量
- ✅ 追求题目质量、深度、准确性
- ✅ 每道题都要经过 AI 深度理解和重新组织
- ✅ 宁缺毋滥，不为了填充而收集

### 2. 原创性整理

- ❌ 禁止原封不动抄袭原文
- ✅ 必须用自己的语言重新组织
- ✅ 添加 AI 的理解和洞察
- ✅ 标注来源，但内容是整理后的

### 3. 内容甄别

- ❌ 不盲从收集到的答案
- ✅ 分析答案的正确性
- ✅ 对比多个来源，取最优解
- ✅ 有争议的内容要标注说明

### 4. 持续优化

- ✅ 相似题目要对比分析
- ✅ 可以更新完善现有题目
- ✅ 不只是创建新题，也要优化旧题
- ✅ 记录更新历史和原因

## 🤖 自动收集流程

### 1. 搜索阶段

**关键词策略：**
- 50+ 中英文关键词轮换
- 包含年份（2025、2026）确保前沿性
- 包含具体技术栈（如 "Next.js AI integration"）

**示例关键词：**
```
"AI interview questions 2026"
"LLM 大模型面试题 2025 2026"
"Agent 多 Agent 协作 工程化"
"前端 AI 应用开发 最佳实践"
"SDD TDD ATDD 开发方法论"
"OpenCode Cursor Windsurf 对比"
"OpenClaw 技能开发 教程"
```

**收集源：**
- GitHub（awesome-* 仓库、技术文章）
- Medium（技术专栏）
- 知乎（专栏、高赞回答）
- 掘金（技术文章）
- SegmentFault
- Reddit（r/MachineLearning、r/learnmachinelearning）
- Hacker News
- 技术博客（个人/公司）
- 论文网站（arXiv、Papers With Code）
- 公众号（如果能访问）

**原则：**
- 每次随机选择 20-30 个不同来源
- 不固定网站，避免单一依赖
- 优先选择 2025-2026 年的内容
- 优先选择高赞/高星/高引用的内容

### 2. 分析阶段

**内容甄别：**
1. 检查答案的正确性（对比多个来源）
2. 识别过时信息（如旧版 API、已淘汰技术）
3. 标注有争议的内容
4. 补充最新进展

**深度整理：**
1. 用自己的语言重新组织
2. 添加代码示例（如果适用）
3. 补充考察重点
4. 生成延伸追问（2-3 个）
5. 添加 AI 的深入理解

**图片处理：**
1. 检测 Markdown 中的图片链接
2. 下载图片到 `/public/images/questions/[日期]/[文件名]`
3. 替换为本地路径
4. 记录到 Frontmatter

### 3. 对比更新阶段

**相似题目检测：**
1. 标题相似度 > 80% → 可能是同一题
2. 内容相似度 > 70% → 可能是变体
3. 标签重合度 > 50% → 可能相关

**更新策略：**
- 如果新内容更准确/更全面 → 更新现有题目
- 如果是不同角度 → 创建新题，互相链接
- 如果内容过时 → 标记为"待更新"
- 如果有错误 → 修正并记录更新历史

**更新流程：**
```
1. 检测相似题目
2. 对比新旧内容
3. 决定：更新 or 新建
4. 如果更新：
   - 合并优点
   - 修正错误
   - 补充新内容
   - 增加版本号
   - 记录更新历史
5. 如果新建：
   - 添加延伸题目链接到相关旧题
```

### 4. 提交部署阶段

**Git 提交规范：**
```
feat: 新增 LLM 题目 - Transformer 架构解析
fix: 修正 CV 题目中的 CNN 公式错误
update: 优化 Agent 题目答案，补充追问
refactor: 重构分类结构
docs: 更新开发指南
```

**自动触发：**
- Git push → GitHub → Vercel 自动部署
- 部署完成后访问 URL 验证

## 📅 定时任务配置

### 执行时间
- **每天凌晨 2 点**（Asia/Shanghai）
- 避开高峰期，减少 API 压力

### 执行内容
1. 运行收集脚本
2. 收集 10-15 道高质量题目（不限制数量，重质量）
3. 分析、整理、去重、对比
4. Git commit + push
5. 触发 Vercel 部署
6. 记录执行日志

### 错误处理
- 网络错误 → 重试 3 次
- API 限流 → 等待后重试
- 内容质量低 → 跳过，记录原因
- Git 冲突 → 拉取最新代码后重试

### 通知机制
- 执行完成 → 记录日志
- 执行失败 → 记录错误，下次重试
- 收集到高质量内容 → 可选项：发送通知
