## 🔄 进化记录

### 2026-04-20 — 第 1 次进化：收集策略升级

**触发事件：** 用户指出 GitHub 收集策略太单一（只看 Trending，遗漏大量优秀项目）

**反思：**
- 过去 10 天只用 GitHub Trending 收集项目
- Trending 只看"近期增长"，会漏掉总星数高但增长平缓的优质项目
- 用户提到的典型案例：很多优秀项目不在 Trending 但 stars 很高
- **根本原因：** 没有主动思考"我的收集方式是否全面"

**策略调整：**
1. 新增 Topics 扫描机制 — 按 AI 相关 topics 搜索高星项目（5000+ stars 门槛）
2. 维护本地 `data/ai-topics.json` AI topics 库（50 个 topics）
3. 发现新项目时提取其 topics，动态更新 topic 库
4. 创建 `scripts/discover-topic-projects.mjs` 自动化扫描脚本

**待验证效果：** 等 20:00 定时任务执行后评估
