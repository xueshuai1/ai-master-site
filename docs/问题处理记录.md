# Mermaid 图表渲染问题修复指南

## 问题描述

在 Next.js + React Markdown 项目中，Mermaid 图表代码块无法正常渲染，只显示为普通代码块。

## 症状

- ✅ 代码高亮正常
- ✅ 表格显示正常
- ❌ Mermaid 图表未渲染（显示为原始代码）
- ❌ 浏览器控制台无相关错误

## 根因分析

1. `react-markdown` 默认不支持 Mermaid 语法
2. `@theguild/remark-mermaid` 插件是为 MDX 设计的，不适用于 `react-markdown`
3. 需要在客户端动态渲染 Mermaid 图表（需要 JavaScript 执行）

## 解决方案

### 方案：创建自定义 MermaidDiagram 组件

#### 1. 创建客户端组件

```tsx
// src/components/MermaidDiagram.tsx
'use client';

import { useEffect, useRef, useState } from 'react';
import mermaid from 'mermaid';

interface MermaidDiagramProps {
  value: string;
}

export default function MermaidDiagram({ value }: MermaidDiagramProps) {
  const containerRef = useRef<HTMLDivElement>(null);
  const [svg, setSvg] = useState<string>('');
  const [error, setError] = useState<string>('');

  useEffect(() => {
    const renderDiagram = async () => {
      try {
        mermaid.initialize({
          startOnLoad: false,
          securityLevel: 'loose',
          theme: 'default',
          fontFamily: 'inherit',
        });

        const { svg: renderedSvg } = await mermaid.render(
          'mermaid-' + Math.random().toString(36).substring(2),
          value
        );
        
        setSvg(renderedSvg);
        setError('');
      } catch (err) {
        console.error('Mermaid render error:', err);
        setError('图表渲染失败：' + (err as Error).message);
      }
    };

    renderDiagram();
  }, [value]);

  if (error) {
    return (
      <div className="my-6 p-4 bg-red-50 border border-red-200 rounded-lg">
        <p className="text-red-600 text-sm">{error}</p>
        <pre className="mt-2 text-xs text-gray-600 overflow-x-auto">
          <code>{value}</code>
        </pre>
      </div>
    );
  }

  return (
    <div ref={containerRef} className="my-6 flex justify-center">
      {svg ? (
        <div dangerouslySetInnerHTML={{ __html: svg }} />
      ) : (
        <div className="text-gray-400">加载中...</div>
      )}
    </div>
  );
}
```

#### 2. 修改文章页面

```tsx
// src/app/knowledge/[category]/[articleId]/page.tsx
import MermaidDiagram from '@/components/MermaidDiagram';

// 在 ReactMarkdown 的 components 中添加自定义 code 渲染
<ReactMarkdown
  remarkPlugins={[remarkGfm]}
  rehypePlugins={[rehypeHighlight]}
  components={{
    code({ node, inline, className, children, ...props }: any) {
      const match = /language-(\w+)/.exec(className || '');
      const content = String(children).replace(/\n$/, '');
      
      // 检测 Mermaid 图表
      if (!inline && match && match[1] === 'mermaid') {
        return <MermaidDiagram value={content} />;
      }
      
      // 普通代码块...
    }
  }}
>
```

## 常见问题

### Q: 为什么不使用 @theguild/remark-mermaid？

A: 该插件是为 MDX 设计的，它会将 Mermaid 代码块转换为 `<Mermaid />` 组件的 MDX 导入。但 `react-markdown` 不支持这种 MDX 特有的转换方式。

### Q: 为什么要用 'use client'？

A: Mermaid 渲染需要访问 DOM 和浏览器 API（如 `document`），这些在服务端渲染时不可用。必须使用客户端组件。

### Q: 图表渲染失败怎么办？

A: 组件内置了错误处理，会显示错误信息和原始代码。常见原因：
- Mermaid 语法错误
- 图表类型不支持
- 特殊字符未转义

## 优化建议

1. **性能优化**: 使用 IntersectionObserver 延迟渲染不可见的图表
2. **主题适配**: 根据系统/用户主题自动切换 Mermaid 主题
3. **缓存**: 缓存已渲染的 SVG，避免重复渲染
4. **SSR 支持**: 可以考虑在服务端预渲染为静态 SVG
