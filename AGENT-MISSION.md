## 📊 工作优先级（从高到低）

### P0 — 知识库持续进化（最高优先级）
**这是网站的核心价值。用户来这里是学习 AI 知识的，不是看新闻的。**

- **发现新方向**：每次巡检主动搜索 AI 前沿（新论文、新技术、新范式），发现有价值的主题就生成知识库文章
  - 来源：arXiv cs.AI/cs.LG/cs.CL 最新论文、OpenAI/Anthropic/Google DeepMind 官方博客、HuggingFace 新模型、GitHub trending AI 项目
  - 标准：这个技术/概念是否重要到值得写一篇文章教别人？如果是，就写
- **完善旧文档**：老文章不能放着发霉
  - LLM/Agent/RAG 等热门领域 → 每月检查是否需要更新（新进展、新数据、新工具）
  - 成熟领域（数学/经典 ML） → 每季度检查
  - 更新方式：不删除原有内容，在相关章节末尾追加「更新于 YYYY-MM-DD」新进展
- **补缺失分类**：检查 KNOWLEDGE-BASE-PLAN.md，发现缺失就补
- **质量标准**：每篇至少 7 章结构（概念→原理→实战→代码→对比→注意事项→扩展阅读），不能是几句话的摘要

### P1 — 工具集紧跟时代
**AI 工具每天冒出来，工具集必须是活的。**

- **发现新工具（两个渠道并行收集）：**
  1. **GitHub Trending**：每周看 trending AI 项目（周增 >1000 stars）
  2. **🔴 GitHub Topics 扫描（每次必做！）**：
     - 运行 `node scripts/discover-topic-projects.mjs`
     - 脚本读取 `data/ai-topics.json`（50 个 AI topics）
     - 对比 tools.ts 已有项目，输出遗漏列表
     - 发现遗漏的高星项目 → 立即收录到 tools.ts
     - 发现新 AI topic → 更新 ai-topics.json
  3. HuggingFace 新模型/新空间
  4. Product Hunt AI 分类
  5. 各公司官方博客（OpenAI、Anthropic、Google、Meta）
- **更新老工具信息**：工具的版本、功能、定价会变，定期更新（至少每季度检查一次主流工具）
- **质量标准**：每个工具至少包含 pros、cons、useCase、learnMore，不能只有一句话 description

## 🧪 QA 自检清单（每次改代码后必须执行）

**这是铁律，不是建议。今天踩过的坑，不能再踩第二次。**

### 移动端必检（390px 视口）
- [ ] **内容横向溢出**：页面不能有横向滚动条（除非是表格/代码块的刻意设计）
  - 重点检查：Markdown 渲染的表格、长 URL、pre 代码块、flex 容器子元素
  - 修复方案：给内容区加 `overflow-x-auto` 或 `min-w-0` 或 `break-all`
- [ ] **文字不截断**：标题、链接、标签在小屏下不能撑破容器
  - 重点检查：长标题、长 URL、标签列表
  - 修复方案：`truncate`、`line-clamp`、`flex-wrap`
- [ ] **按钮/链接可点击**：元素不能重叠、不能被其他元素遮挡

### PC 端必检（1440px 视口）
- [ ] 布局正常，内容不溢出
- [ ] 导航链接都能跳转到正确页面
- [ ] 分页功能正常

### 交互必检
- [ ] **快速点击不崩溃**：分类筛选、排序按钮等高频交互，快速连续点击不能导致状态混乱
  - 常见原因：`router.replace` 竞态、防抖缺失、状态重置不完整
  - 修复方案：对 URL 同步加防抖（400ms），筛选变化时自动重置页码
- [ ] **数据一致性**：同一份数据在不同页面的统计数字必须一致
  - 常见原因：不同页面用了不同的筛选/排序逻辑
  - 修复方案：提取共用函数，不要每个页面自己算

### 代码规范
- [ ] 所有新增/修改的文字内容必须是中文
- [ ] `build` 通过才能提交
- [ ] 首页时间戳同步更新
- [ ] 🔴 **转义字符检查**：tip/warning/body 中不能出现 `\\n` 字面量（应使用实际换行或 `<br>`）
- [ ] 🔴 **HTML 标签检查**：Mermaid/表格中不能出现未转义的 HTML 标签（如 `<br>` 在 Mermaid 中会导致渲染失败）
