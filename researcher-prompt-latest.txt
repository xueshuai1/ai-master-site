
### 4B. 写新新闻
每条新闻格式：
```typescript
{
  id: "news-{序号}",
  tag: "分类标签",        // 大语言模型/Agent/开源项目/芯片/应用/政策/行业
  title: "标题（中文）",
  summary: "2-3 句话摘要",
  content: "详细内容（markdown 格式，可分段）",
  date: "YYYY-MM-DD HH:MM",
  source: "来源（如：OpenAI Blog + 今日头条）",
  sourceUrl: "原始链接",
  href: "/news/news-{序号}",
}
```

**新闻要求：**
- 标题和正文必须是中文
- ⛔ **禁止在 summary 和 content 字符串中使用中文弯引号（""）、破折号（——）等可能破坏 TS 语法的字符！必须用直引号（""）或中文方括号【】替代！**
- 同一事件被多个来源报道 → 合并为一条，列出各方观点
- source 字段写多个来源（如："arXiv + 机器之心 + Hacker News"）
- 必须是外部真实事件，不是本站自宣
- 写完后查重：grep -i '关键词' src/data/news.ts

### 4C. 校验
```bash
node scripts/validate-data.mjs 2>&1 || echo "跳过"
```

### 4D. ⛔ 编译校验（提交前必须通过！不可跳过！）
写完 news.ts / tools.ts 等数据文件后，必须跑编译验证：
```bash
npm run build 2>&1 | tail -20
npx tsc --noEmit 2>&1 | tail -10
```
- **编译通过才能进入第 6 步！失败 = 立即修复，禁止提交推送！**
- 常见错误：字符串中的中文弯引号（``""``）会被 TS 解析器识别为字符串结束符，导致 Syntax Error
- 修复后重新跑 build，直到通过才继续
