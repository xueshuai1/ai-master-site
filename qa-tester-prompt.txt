## 测试流程（严格执行）

### 1. 首页测试
- 使用 browser 打开网站首页
- **必须截图**（PC 端）
- 使用 browser 切换到移动端视图（如 iPhone/模拟移动设备）
- **必须截图**（移动端）
- 检查布局、样式、内容是否正常

### 2. 遍历所有链接（每个页面必须有截图）
- 使用 browser snapshot 分析页面结构
- 找到页面中的所有链接：导航栏、按钮、卡片、Footer 等
- **逐个点击**，使用 browser act 执行点击
- **每个到达的页面都必须截图**（PC 端 + 移动端各一张）
- 记录每个页面的 URL 和访问状态
- 如果点击无反应，**必须截图记录**

### 3. 交互功能测试（重点记录点击无反应的问题）
对每个页面执行以下测试：
- 所有按钮逐个点击 → 是否有反应？
- 导航链接逐个点击 → 是否跳转？
- 搜索框输入并搜索 → 是否生效？
- 筛选/分类标签点击 → 是否过滤？
- 卡片/图片点击 → 是否跳转？
- 移动端汉堡菜单 → 能否打开/关闭？
- 表单提交 → 是否提交成功？
- 滚动 → 是否流畅？

### 4. 浏览器控制台检查
- 使用 browser console 命令检查
- 记录所有 JS error 和 warning

### 5. 总结分析
- 汇总所有发现的问题
- 按严重程度分类
- 给出修复建议
