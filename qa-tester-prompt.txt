## 🔴 测试流程（严格执行，不可跳过任何步骤）

### 第一阶段：全局扫描（5 分钟）

**1. 首页快速浏览**
```
打开 https://ai-master.cc（或本地开发服务器）
截图：PC 端 + 移动端
检查：首屏内容是否正常？导航是否完整？有没有明显异常？
```

**2. 站点地图扫描**
```
用 browser snapshot 分析首页
记录所有可点击的链接：导航栏、卡片、按钮、Footer、侧边栏
确认所有页面路径：/ /news /blog /knowledge /tools /roadmap /about 等
```

### 第二阶段：逐页深度测试（每个页面必须完成以下全部检查）

对每个页面执行：

**1. 可访问性检查**
- 页面能否正常加载？
- 有没有白屏、404、500 错误？
- 加载时间是否合理（<3 秒）？

**2. PC 端截图 + 分析**
- 全屏截图
- 检查：布局、字体、颜色、间距、图片、卡片效果

**3. 移动端截图 + 分析**
- 切换到移动设备视图（iPhone 12/14 或 375px 宽度）
- 截图
- 检查：
  - 导航是否变为汉堡菜单？
  - 内容是否自适应宽度？
  - 文字是否溢出？
  - 按钮是否容易点击（不小于 44x44px）？
  - 横向滚动条是否不应该出现？
- **🔴 横向溢出专项检查（2026-04-20 教训）：**
  - 在移动端视口下从左到右滑动整个页面
  - 重点检查：Markdown 渲染的表格、长 URL、pre 代码块、长标题、flex 子元素
  - 任何不应出现横向滚动的区域都算 P0 bug
  - 表格、代码块等刻意设计的横向滚动除外，但也必须有 overflow-x-auto 容器

**4. 控制台检查**
```
browser console 命令
记录所有 JS error 和 warning
特别注意：
- Uncaught TypeError
- Failed to load resource
- 404 错误
- React warning
```

**5. 交互功能逐项测试**

| 功能类别 | 测试项 | 状态 |
|---------|--------|------|
| 导航 | 所有导航链接是否跳转正确？ | ✅/❌ |
| 卡片 | 新闻/博客/知识卡片点击是否跳转？ | ✅/❌ |
| 按钮 | 所有按钮点击是否有反应？ | ✅/❌ |
| 筛选 | 分类/标签筛选是否生效？**快速连续点击是否导致状态混乱？** | ✅/❌ |
| 搜索 | 搜索框输入+提交是否正常？ **快速输入是否导致异常？** | ✅/❌ |
| 分页 | 分页按钮是否工作？ | ✅/❌ |
| 滚动 | 页面滚动是否流畅？ | ✅/❌ |
| 动画 | 动画是否正常播放/有无闪烁？ | ✅/❌ |

**6. 内容质量抽查**
- 文章是否有真实内容？（不是只有标题）
- 代码块是否渲染正确？（应该有语法高亮，不是纯文本）
- 图片是否加载正常？
- 链接是否有效（不是死链）？
- 🔴 **渲染质量专项检查（2026-04-28 新增）：**
  - 打开任意一篇最近修改的知识库/博客文章
  - 检查 Mermaid 图表是否正常渲染（不是空白块/黑块/文字不可见）
  - 检查代码块是否有语法高亮（不是纯文本代码框）
  - 检查页面是否有异常的空白区域
  - 检查 Mermaid 图表中节点文字是否清晰可读
- 🔴 **系列一致性检查（2026-05-09 新增）：**
  - 打开有系列标签的文章，检查「系列上一篇/下一篇」导航是否正常显示
  - 检查系列文章的标题是否包含编号（一）（二）（三）
  - 点击系列导航链接，确认上下篇跳转正确
  - 检查学习路线页面是否按系列顺序排列文章

### 第三阶段：回归测试

- 检查上次 QA 报告中的问题是否已修复
- 记录未修复的问题
- 确认新改动没有引入新问题
