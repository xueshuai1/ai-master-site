1. RAG 的基本架构和工作流程是什么？

3. 检索器和生成器如何协同工作？
4. RAG 的典型应用场景有哪些？
5. RAG 相比纯生成式模型有什么优势和劣势？

## 参考答案

### 1. RAG 基本架构

RAG 由两个核心组件组成：

**检索器（Retriever）**：
- 从外部知识库中检索相关文档
- 通常使用向量数据库（如 FAISS、Pinecone、Weaviate）
- 将查询转换为向量，计算相似度

**生成器（Generator）**：
- 基于检索到的文档生成回答
- 通常使用 LLM（如 GPT、Claude、LLaMA）
- 将检索内容作为上下文输入

**工作流程**：
```
用户查询 → 向量化 → 检索相关文档 → 拼接上下文 → LLM 生成 → 输出回答
```

### 2. RAG 解决的问题

**LLM 的局限性**：
- **知识截止**：训练数据有时间限制，无法获取最新信息
- **幻觉问题**：可能生成看似合理但错误的内容
- **私有知识**：无法访问企业私有数据
- **可解释性差**：难以追溯答案来源

**RAG 的优势**：
- ✅ 实时获取最新信息
- ✅ 减少幻觉，提高准确性
- ✅ 访问私有知识库
- ✅ 提供引用来源，增强可解释性

### 3. 检索器与生成器协同

**检索阶段**：
```python
# 1. 将用户查询向量化
query_embedding = embedder.encode(query)

# 2. 在向量数据库中检索
results = vector_db.search(query_embedding, top_k=5)

# 3. 获取相关文档
contexts = [doc.content for doc in results]
```

**生成阶段**：
```python
# 构建 prompt
prompt = f"""基于以下信息回答问题：

相关信息：
{' '.join(contexts)}

问题：{query}

回答："""

# 调用 LLM
response = llm.generate(prompt)
```

### 4. 典型应用场景

| 场景 | 描述 | 示例 |
|------|------|------|
| **企业知识库问答** | 基于内部文档回答问题 | 员工手册、技术文档查询 |
| **客服机器人** | 结合产品文档回答用户问题 | 电商、SaaS 产品客服 |
| **法律/医疗咨询** | 基于法规/文献提供建议 | 法律条文查询、医学文献 |
| **研究助手** | 基于论文库回答问题 | 学术文献检索和总结 |
| **代码助手** | 基于代码库回答编程问题 | 内部 API 文档查询 |

### 5. 优势与劣势

**优势**：
- ✅ 知识可更新，无需重新训练
- ✅ 减少幻觉，提高可信度
- ✅ 支持私有数据
- ✅ 可追溯答案来源
- ✅ 成本较低（相比微调）

**劣势**：
- ❌ 依赖检索质量
- ❌ 上下文长度限制
- ❌ 检索延迟影响响应时间
- ❌ 复杂查询可能检索不准确
- ❌ 需要维护向量数据库

## 延伸追问

### 追问 1: 如果检索到的文档与问题不相关，会导致什么问题？如何优化？

**答案：**
会导致 LLM 基于错误信息生成回答（"垃圾进，垃圾出"）。

**优化方案**：
1. **改进检索**：使用更好的 embedding 模型、混合检索（关键词 + 向量）
2. **重排序**：对检索结果进行二次排序（如使用 Cross-Encoder）
3. **查询改写**：扩展或改写用户查询
4. **设置阈值**：相似度低于阈值时返回"未找到相关信息"
5. **多路召回**：从多个来源检索，然后融合结果

### 追问 2: RAG 系统中如何处理多轮对话的上下文？

**答案：**
多轮对话的 RAG 需要特殊处理：

1. **历史对话检索**：将对话历史也纳入检索范围
2. **查询改写**：基于对话历史改写当前查询（解决指代问题）
3. **上下文窗口管理**：合理分配 token 给历史对话和检索文档
4. **会话级向量**：为整个会话生成向量，而不仅是单轮查询

```python
# 查询改写示例
rewritten_query = llm.generate(f"""
基于对话历史，改写当前问题使其独立完整：

对话历史：{conversation_history}
当前问题：{current_query}

改写后的问题：""")
```

### 追问 3: 在企业环境中部署 RAG 系统，需要考虑哪些工程问题？

**答案：**

**性能方面**：
- 检索延迟优化（缓存、索引优化）
- 并发处理能力
- 响应时间 SLA

**安全方面**：
- 文档访问权限控制
- 敏感信息过滤
- 审计日志

**运维方面**：
- 知识库更新流程
- 向量数据库备份和恢复
- 监控和告警（检索质量、生成质量）

**成本方面**：
- LLM API 调用成本优化
- 向量数据库存储成本
- 缓存策略降低重复查询

## 深入理解

> **实际应用场景**：
> - **电商客服**：结合商品文档、订单信息回答用户问题
> - **企业内部助手**：基于内部文档、代码库、会议记录提供信息
> - **教育领域**：基于教材、 lecture notes 回答学生问题
>
> **与其他知识点的关联**：
> - **向量数据库**：FAISS、Pinecone、Chroma 等
> - **Embedding 模型**：text-embedding-ada-002、BGE、M3E 等
> - **LLM 上下文学习**：In-context Learning 原理
> - **Agent 系统**：RAG 可作为 Agent 的工具之一
>
> **进阶变体**：
> - **Advanced RAG**：查询改写、多路召回、重排序
> - **Modular RAG**：模块化设计，灵活组合检索和生成策略
> - **Self-RAG**：模型自主判断是否需要检索
> - **Corrective RAG**：自动评估和修正检索结果
>
