## 题目描述
请解释集成学习 (Ensemble Learning) 的核心思想。Bagging 和 Boosting 有什么区别？请详细说明 Random Forest、XGBoost、LightGBM 的工作原理和适用场景。

## 参考答案

### 集成学习的核心思想

**"三个臭皮匠，顶个诸葛亮"** —— 通过组合多个弱学习器，获得比单个模型更好的泛化性能。

**为什么有效：**
1. **减少方差** (Bagging): 平均多个模型的预测
2. **减少偏差** (Boosting): 逐步修正错误
3. **提高稳定性**: 降低对单个模型的依赖

### Bagging vs Boosting 对比

| 特性 | Bagging | Boosting |
|------|---------|----------|
| **全称** | Bootstrap Aggregating | Adaptive Boosting |
| **训练方式** | 并行训练 | 串行训练 |
| **样本采样** | 有放回随机采样 | 关注之前预测错误的样本 |
| **模型权重** | 所有模型平等投票 | 根据性能加权 |
| **主要目标** | 减少方差 | 减少偏差 |
| **过拟合风险** | 较低 | 较高（需要早停） |
| **典型算法** | Random Forest | AdaBoost, XGBoost, LightGBM |

### 1. Random Forest (Bagging 代表)

```python
from sklearn.ensemble import RandomForestClassifier, RandomForestRegressor
from sklearn.model_selection import cross_val_score
import numpy as np

# 随机森林分类器
rf_clf = RandomForestClassifier(
    n_estimators=100,      # 树的数量
    max_depth=20,          # 树的最大深度
    min_samples_split=5,   # 内部节点再划分所需最小样本数
    min_samples_leaf=2,    # 叶子节点最小样本数
    max_features='sqrt',   # 特征采样：'sqrt', 'log2', 或 float
    bootstrap=True,        # 是否使用 bootstrap 采样
    oob_score=True,        # 是否使用 OOB 样本评估
    n_jobs=-1,             # 并行计算
    random_state=42,
    class_weight='balanced' # 处理不平衡数据
)

rf_clf.fit(X_train, y_train)

# OOB 分数 (无需额外验证集)
print(f"OOB Score: {rf_clf.oob_score_:.4f}")

# 特征重要性
importances = rf_clf.feature_importances_
top_features = np.argsort(importances)[-10:][::-1]
print(f"Top 10 features: {top_features}")

# 可视化特征重要性
import matplotlib.pyplot as plt
plt.figure(figsize=(10, 6))
plt.barh(range(len(top_features)), importances[top_features])
plt.yticks(range(len(top_features)), top_features)
plt.xlabel('Importance')
plt.title('Random Forest Feature Importances')
plt.gca().invert_yaxis()
plt.tight_layout()
plt.show()
```

**工作原理：**
1. Bootstrap 采样：从原始数据中有放回地抽取 n 个样本
2. 随机特征：每次分裂只考虑随机子集的特征
3. 多棵树：重复以上步骤构建多棵决策树
4. 投票/平均：分类用投票，回归用平均

**优点：**
- 不容易过拟合
- 可以处理高维数据
- 提供特征重要性
- 对异常值不敏感

**缺点：**
- 模型较大，预测较慢
- 可解释性不如单棵树
- 对于线性关系建模能力弱

### 2. XGBoost (Boosting 代表)

```python
import xgboost as xgb
from sklearn.model_selection import GridSearchCV

# XGBoost 分类器
xgb_clf = xgb.XGBClassifier(
    n_estimators=100,          # 树的数量
    max_depth=6,               # 树的最大深度
    learning_rate=0.1,         # 学习率 (eta)
    subsample=0.8,             # 样本采样比例
    colsample_bytree=0.8,      # 特征采样比例
    min_child_weight=1,        # 子树最小权重和
    gamma=0,                   # 分裂所需最小损失减少
    reg_alpha=0,               # L1 正则化
    reg_lambda=1,              # L2 正则化
    scale_pos_weight=1,        # 处理不平衡数据
    objective='binary:logistic',
    eval_metric='logloss',
    tree_method='hist',        # 树构建算法
    n_jobs=-1,
    random_state=42
)

# 使用早停防止过拟合
xgb_clf.fit(
    X_train, y_train,
    eval_set=[(X_val, y_val)],
    early_stopping_rounds=50,
    verbose=True
)

# 最佳迭代次数
print(f"Best iteration: {xgb_clf.best_iteration}")

# 特征重要性
xgb.plot_importance(xgb_clf, max_num_features=20)
plt.show()
```

**工作原理：**
1. 加法模型：逐步添加新树修正残差
2. 梯度提升：用负梯度方向拟合残差
3. 正则化：目标函数包含正则项
4. 二阶泰勒展开：利用一阶和二阶导数

**目标函数：**
```
Obj(θ) = Σ l(y_i, ŷ_i) + Σ Ω(f_k)
       = Σ l(y_i, ŷ_i) + Σ (γT + ½λ||w||²)
```

**优点：**
- 预测精度高
- 支持多种目标函数
- 内置正则化
- 支持缺失值处理

**缺点：**
- 参数较多，调参复杂
- 容易过拟合（需要早停）
- 训练时间较长

### 3. LightGBM (高效 Boosting)

```python
import lightgbm as lgb

# LightGBM 分类器
lgb_clf = lgb.LGBMClassifier(
    n_estimators=100,
    max_depth=-1,              # 无限制（用 num_leaves 控制）
    num_leaves=31,             # 叶子节点数
    learning_rate=0.1,
    subsample=0.8,
    colsample_bytree=0.8,
    min_child_samples=20,      # 叶子最小样本数
    reg_alpha=0.1,
    reg_lambda=0.1,
    class_weight='balanced',
    objective='binary',
    metric='binary_logloss',
    n_jobs=-1,
    random_state=42,
    verbose=-1
)

lgb_clf.fit(
    X_train, y_train,
    eval_set=[(X_val, y_val)],
    eval_metric='binary_logloss',
    callbacks=[
        lgb.early_stopping(stopping_rounds=50),
        lgb.log_evaluation(period=50)
    ]
)

# 特征重要性
lgb.plot_importance(lgb_clf, max_num_features=20)
plt.show()
```

**核心创新：**
1. **GOSS** (Gradient-based One-Side Sampling): 保留大梯度样本
2. **EFB** (Exclusive Feature Bundling): 捆绑互斥特征
3. **Leaf-wise 生长**: 按叶子分裂，而非按层

**优点：**
- 训练速度极快
- 内存占用低
- 支持大规模数据
- 精度高

**缺点：**
- 小数据可能过拟合
- 参数敏感

### 4. Stacking (堆叠集成)

```python
from sklearn.ensemble import StackingClassifier, VotingClassifier
from sklearn.linear_model import LogisticRegression
from sklearn.svm import SVC
from sklearn.neighbors import KNeighborsClassifier
from sklearn.naive_bayes import GaussianNB

# 基学习器
base_estimators = [
    ('rf', RandomForestClassifier(n_estimators=50, random_state=42)),
    ('xgb', xgb.XGBClassifier(n_estimators=50, random_state=42)),
    ('svm', SVC(probability=True, random_state=42)),
    ('knn', KNeighborsClassifier(n_neighbors=5))
]

# 元学习器
meta_estimator = LogisticRegression()

# Stacking
stacking_clf = StackingClassifier(
    estimators=base_estimators,
    final_estimator=meta_estimator,
    cv=5,                      # 交叉验证生成元特征
    stack_method='predict_proba',  # 使用概率作为元特征
    n_jobs=-1
)

stacking_clf.fit(X_train, y_train)
score = stacking_clf.score(X_test, y_test)
print(f"Stacking Accuracy: {score:.4f}")

# 对比各模型性能
print("\nIndividual Models:")
for name, clf in base_estimators:
    clf.fit(X_train, y_train)
    score = clf.score(X_test, y_test)
    print(f"  {name}: {score:.4f}")
```

**工作原理：**
1. 第一层：多个基学习器训练
2. 第二层：用基学习器的预测作为输入，训练元学习器
3. 交叉验证：避免过拟合（用 OOF 预测训练元学习器）

## 延伸追问

### 追问 1: Random Forest 中为什么要随机选择特征？

**问题：** 随机特征子集的作用是什么？

**答案：**

```python
# 演示随机特征的效果
from sklearn.tree import DecisionTreeClassifier
from sklearn.ensemble import BaggingClassifier

# 没有随机特征的 Bagging
bagging_no_feat = BaggingClassifier(
    DecisionTreeClassifier(),
    n_estimators=100,
    max_features=1.0,  # 使用所有特征
    random_state=42
)

# 有随机特征的 Random Forest
rf = RandomForestClassifier(
    n_estimators=100,
    max_features='sqrt',  # 随机选择 sqrt(n_features)
    random_state=42
)

# 随机特征的作用：
# 1. 增加树之间的差异性（去相关）
# 2. 减少方差（集成效果更好的前提是模型独立）
# 3. 对于高维数据特别有效

# 方差减少公式：
# Var(average) = ρ * σ²
# 其中 ρ 是树之间的相关系数
# 随机特征降低了 ρ，从而减少了整体方差
```

### 追问 2: XGBoost 的正则化项有什么作用？

**问题：** XGBoost 目标函数中的正则项如何防止过拟合？

**答案：**

```python
# XGBoost 目标函数
# Obj = Σ l(y_i, ŷ_i) + Σ Ω(f_k)
# Ω(f) = γT + ½λ||w||²

# γ (gamma): 控制树的复杂度
# - 每次分裂需要损失减少 > γ
# - γ 越大，树越简单

# λ (lambda): L2 正则化叶子权重
# - 惩罚大的叶子权重
# - 防止某个叶子主导预测

# α (alpha): L1 正则化叶子权重
# - 产生稀疏的叶子权重
# - 类似 Lasso 的特征选择效果

# 调参建议
param_grid = {
    'gamma': [0, 0.1, 0.2, 0.5],
    'reg_lambda': [1, 5, 10, 20],
    'reg_alpha': [0, 0.1, 0.5, 1]
}

# 过拟合时：增加 γ, λ, α
# 欠拟合时：减小 γ, λ, α
```

### 追问 3: 如何选择基学习器进行 Stacking？

**问题：** Stacking 中基学习器的选择有什么原则？

**答案：**

```python
# 选择原则：
# 1. 多样性：不同类型的模型（树、线性、距离、概率）
# 2. 互补性：各模型在不同样本上表现不同
# 3. 性能：每个基学习器都应有一定性能

# 好的组合示例
good_ensemble = [
    ('tree', RandomForestClassifier()),      # 树模型
    ('boost', xgb.XGBClassifier()),          # 提升树
    ('linear', LogisticRegression()),        # 线性模型
    ('distance', KNeighborsClassifier()),    # 距离模型
    ('prob', GaussianNB())                   # 概率模型
]

# 不好的组合（太相似）
bad_ensemble = [
    ('rf1', RandomForestClassifier()),
    ('rf2', RandomForestClassifier()),
    ('rf3', RandomForestClassifier())
]

# 评估多样性
from sklearn.metrics import pairwise_distances

# 训练各模型并获取预测
predictions = []
for name, clf in good_ensemble:
    clf.fit(X_train, y_train)
    pred = clf.predict_proba(X_val)[:, 1]
    predictions.append(pred)

predictions = np.array(predictions).T

# 计算预测相关性
corr_matrix = np.corrcoef(predictions.T)
print("Prediction Correlation Matrix:")
print(corr_matrix)
# 理想情况：相关性低（模型预测不同）
```

## 深入理解

**集成学习成功的关键：**

1. **准确性**：每个基学习器都要比随机猜测好
2. **多样性**：基学习器之间的错误要独立
3. **平衡**：偏差和方差的权衡

**实际应用建议：**
- 从 Random Forest 开始（简单有效）
- 追求精度用 XGBoost/LightGBM
- 比赛/关键任务用 Stacking
- 注意计算成本和推理时间
