## 题目描述
什么是数据泄露 (Data Leakage)？有哪些常见类型？如何检测和预防数据泄露？

## 参考答案

### 数据泄露定义

**数据泄露：** 训练时使用了测试时不可用的信息，导致评估过于乐观，生产性能下降。

### 常见类型

```python
from sklearn.model_selection import train_test_split
from sklearn.preprocessing import StandardScaler
from sklearn.pipeline import Pipeline
from sklearn.ensemble import RandomForestClassifier

# ❌ 类型 1: 预处理泄露
# 在划分前进行标准化
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)  # 用了全部数据！
X_train, X_test = train_test_split(X_scaled, test_size=0.2)  # 泄露！

# ✅ 正确做法
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2)
scaler = StandardScaler()
X_train_scaled = scaler.fit_transform(X_train)  # 只用训练数据拟合
X_test_scaled = scaler.transform(X_test)        # 用训练数据的参数转换

# ✅ 最佳做法：Pipeline
pipeline = Pipeline([
    ('scaler', StandardScaler()),
    ('clf', RandomForestClassifier())
])
pipeline.fit(X_train, y_train)  # 自动正确处理
```

```python
# ❌ 类型 2: 特征工程泄露
# 用全部数据计算特征统计量
global_mean = X['feature'].mean()  # 包含测试集！
X['feature_imputed'] = X['feature'].fillna(global_mean)

# ✅ 正确做法
train_mean = X_train['feature'].mean()
```

```python
# 用未来数据预测过去

# ✅ 正确做法：时间序列交叉验证
```

### 检测方法

```python
# 1. 检查训练 - 测试性能差距
train_score = model.score(X_train, y_train)
test_score = model.score(X_test, y_test)
gap = train_score - test_score

if gap > 0.1:
    print(f"⚠️  警告：性能差距 {gap:.4f}，可能存在泄露")

# 2. 特征重要性分析
# 异常重要的特征可能泄露

# 3. 打乱目标验证
from sklearn.model_selection import cross_val_score
import numpy as np

y_shuffled = np.random.permutation(y)
score_shuffled = cross_val_score(model, X, y_shuffled, cv=5).mean()

if score_shuffled > 0.6:  # 应该接近随机
    print(f"⚠️  警告：打乱目标后仍有 {score_shuffled:.4f} 准确率，存在泄露")

# 4. 时间验证
# 用未来数据训练，过去数据测试（应该很差）
```

### 预防清单

```python
prevention_checklist = """
数据泄露预防清单：

□ 训练/测试划分后再预处理
□ 使用 Pipeline 封装预处理
□ 交叉验证中每折独立预处理
□ 时间序列不用未来数据
□ 特征不包含目标信息
□ 分组数据保持组完整
□ 缺失值填充只用训练统计量
□ 特征选择只用训练数据
□ 超参数调优用验证集或嵌套 CV
□ 最终评估用独立测试集
"""
print(prevention_checklist)
```
