# AI Master Site 每日维护计划

> 制定时间：2026-04-12 | 负责人：奥利奥 🍪 | 模式：全自动

---

## 📋 核心原则

1. **新鲜度** — 每日更新，不堆砌，宁缺毋滥
2. **技术深度** — 深入分析再生成，不是简单搬运
3. **前沿性** — 追踪最新论文、工具、行业动态
4. **质量把控** — 生成后必须编译验证，不能改坏已有内容
5. **自主运营** — 我（奥利奥）全权决定更新策略和节奏
6. **🔴 语言要求** — 所有内容（新闻、博客、知识库、工具）必须是中文。信息来源可以是英文网站，但生成的内容必须翻译成中文后写入。禁止直接写入英文。

---

## 🔄 每日更新内容

### 1. 首页新闻（Latest News）
- **频率：** 每日
- **来源：** 主流 AI 新闻网站、Twitter/X、arXiv、Hacker News
- **流程：** 收集 → 筛选 Top5 → 生成首页新闻条目 → 更新
- **文件：** `src/app/page.tsx`（news 数组）

### 2. 博客/行业洞察
- **频率：** 每 2-3 天一篇深度文章
- **来源：** arXiv 最新论文、技术博客、产品发布
- **流程：** 选题 → 深度分析 → 生成文章 → 编译验证 → 提交
- **目录：** `src/app/blog/` + `src/data/blogs.ts`（如果存在）

### 3. 知识库维护
- **频率：** 按需扩展新主题
- **来源：** 新出现的技术方向、用户需求
- **流程：** 评估是否需要新文章 → 生成（7 章格式）→ 编译 → 更新 knowledge.ts → 提交
- **目录：** `src/data/articles/`

### 4. 工具集
- **频率：** 每周更新
- **来源：** GitHub trending、HuggingFace、新产品发布
- **流程：** 发现新工具 → 试用/分析 → 更新工具页面
- **文件：** `src/app/tools/page.tsx`

### 5. 前沿动态（行业洞察）
- **频率：** 每日跟踪，每周汇总
- **来源：** arXiv daily、OpenAI blog、Anthropic blog、Google AI blog
- **流程：** 每日扫描 → 标记重要动态 → 周末汇总成洞察文章

### 6. 学习路径
- **频率：** 按需更新（新技术出现时）
- **来源：** 行业趋势、技术栈变化
- **文件：** `src/app/roadmap/page.tsx`

### 7. 实践教程
- **频率：** 每 1-2 周一篇
- **来源：** 实际项目经验、工具使用心得
- **流程：** 选题 → 实操 → 写教程 → 提交

## ⏰ Cron 定时任务

### 每日任务（每天执行）
1. **08:00** — 扫描 AI 新闻源，更新首页新闻
2. **10:00** — 检查 arXiv 新论文，标记重要论文
3. **20:00** — 汇总当日重要动态

### 每周任务
1. **周一 09:00** — 检查工具集是否需要更新
2. **周三 14:00** — 生成一篇深度博客/洞察文章
3. **周五 16:00** — 检查知识库是否需要扩展
4. **周日 20:00** — 本周总结，规划下周更新

---

## 🛡️ 质量保障

1. **编译验证** — 每次更新后必须 `npx tsc --noEmit` + `npm run build`
2. **已有内容保护** — 不修改已提交的文章文件，只新增
3. **回滚机制** — 更新失败时 git checkout 恢复
4. **更新时间** — 每次内容更新后运行 `scripts/update-time.sh`
