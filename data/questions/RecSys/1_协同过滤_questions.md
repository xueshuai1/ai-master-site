# 协同过滤 - 面试题（5 道）

## 题目 1：协同过滤基础原理

**问题：** 请详细解释协同过滤的核心思想，并说明 User-Based CF 和 Item-Based CF 的区别及各自适用场景。

**参考答案：**

**核心思想：**
协同过滤的核心是"物以类聚，人以群分"，利用用户群体的集体行为数据进行推荐。基本假设：
1. 用户偏好具有稳定性：过去喜欢的物品，未来也可能喜欢
2. 相似用户有相似偏好：如果两个用户历史行为相似，未来行为也可能相似
3. 相似物品被相似用户喜欢：如果两个物品被同一组用户喜欢，它们可能相似

**User-Based vs Item-Based 对比：**

| 维度 | User-Based CF | Item-Based CF |
|------|---------------|---------------|
| 相似度计算对象 | 用户之间 | 物品之间 |
| 推荐逻辑 | 找相似用户，推荐他们喜欢的物品 | 找与历史物品相似的物品 |
| 计算复杂度 | O(U²)，用户数增长时恶化 | O(I²)，物品数增长时恶化 |
| 实时性 | 用户行为变化需重新计算 | 物品关系相对稳定 |
| 可解释性 | "和你相似的用户也喜欢" | "和你看过的物品相似" |
| 适用场景 | 用户少、物品多、用户兴趣变化快 | 物品稳定、用户多（如电商） |

**工业界选择：**
- Amazon 选择 Item-Based：物品库相对稳定，用户数亿级
- 社交网络可能用 User-Based：用户关系网络重要

- 良好（4 分）：理解基本原理，能说明主要区别
- 一般（3 分）：知道概念但对比不清晰
- 较差（≤2 分）：概念混淆或理解错误

## 题目 2：相似度计算

**问题：** 协同过滤中常用的相似度计算方法有哪些？请写出余弦相似度和皮尔逊相关系数的公式，并说明它们的区别和适用场景。

**参考答案：**

**常用相似度方法：**

1. **余弦相似度（Cosine Similarity）**
   $$\text{sim}(u, v) = \frac{\vec{u} \cdot \vec{v}}{||\vec{u}|| \cdot ||\vec{v}||} = \frac{\sum_{i \in I_{uv}} r_{ui} \cdot r_{vi}}{\sqrt{\sum_{i \in I_u} r_{ui}^2} \cdot \sqrt{\sum_{i \in I_v} r_{vi}^2}}$$

2. **皮尔逊相关系数（Pearson Correlation）**
   $$\text{sim}(u, v) = \frac{\sum_{i \in I_{uv}} (r_{ui} - \bar{r}_u)(r_{vi} - \bar{r}_v)}{\sqrt{\sum_{i \in I_u} (r_{ui} - \bar{r}_u)^2} \cdot \sqrt{\sum_{i \in I_v} (r_{vi} - \bar{r}_v)^2}}$$

3. **杰卡德相似度（Jaccard）**：适用于隐式反馈
   $$\text{sim}(u, v) = \frac{|I_u \cap I_v|}{|I_u \cup I_v|}$$

4. **调整余弦相似度**：考虑用户评分偏置

**核心区别：**

| 特性 | 余弦相似度 | 皮尔逊相关系数 |
|------|-----------|---------------|
| 是否中心化 | 否，直接用原始评分 | 是，减去用户平均分 |
| 对用户偏置敏感 | 是（评分尺度不同的用户会被认为不相似） | 否（消除用户评分习惯差异） |
| 计算复杂度 | 较低 | 较高（需计算均值） |
| 适用场景 | 评分标准统一的数据 | 用户评分习惯差异大的数据 |

**示例说明：**
- 用户 A 习惯打高分（4-5 分），用户 B 习惯打低分（1-2 分）
- 但两人对物品的相对偏好一致（都认为物品 1>物品 2>物品 3）
- 余弦相似度会认为两人不相似
- 皮尔逊会识别出两人偏好模式相似

- 良好（4 分）：公式正确，能说明主要区别
- 一般（3 分）：知道公式但不理解差异

## 题目 3：冷启动问题

**问题：** 协同过滤面临哪些冷启动问题？针对新用户、新物品、新系统分别有什么解决方案？

**参考答案：**

**冷启动类型：**

1. **新用户冷启动**：用户无历史行为，无法计算相似度
2. **新物品冷启动**：物品无被交互记录，无法被推荐
3. **新系统冷启动**：系统初期用户和物品都少，数据稀疏

**解决方案：**

**新用户冷启动：**
```
┌─────────────────────────────────────────────────┐
│  1. 注册时收集显式偏好（选择兴趣标签）          │
│  2. 引导式交互（让新用户评分一些物品）          │
│  3. 基于内容推荐（用物品属性匹配）              │
│  4. 热门推荐（推荐流行物品作为起点）            │
│  5. 人口统计学推荐（年龄、性别、地区等）        │
│  6. 社交关系导入（关注的人喜欢什么）            │
└─────────────────────────────────────────────────┘
```

**新物品冷启动：**
```
┌─────────────────────────────────────────────────┐
│  1. 基于内容推荐（用物品元数据）                │
│  2. 主动曝光策略（给一定流量测试）              │
│  3. 利用物品属性找相似老物品                    │
│  4. 上下文感知（时间、地点等）                  │
│  5. 探索与利用（Bandit 算法）                    │
└─────────────────────────────────────────────────┘
```

**新系统冷启动：**
```
┌─────────────────────────────────────────────────┐
│  1. 混合推荐（CF + 内容 + 规则）                │
│  2. 迁移学习（从其他领域/平台迁移）             │
│  3. 主动学习（选择最有信息量的物品让用户评）    │
│  4. 众包/专家标注（人工构建初始关系）           │
│  5. 利用外部知识图谱                            │
└─────────────────────────────────────────────────┘
```

**实际案例：**
- Netflix：新用户选择喜欢的类型和影片
- 抖音：新用户使用热门视频 + 快速滑动收集偏好
- 淘宝：新用户基于浏览行为实时调整

**评分：**
- 优秀（5 分）：分类清晰，方案全面，有实际案例
- 良好（4 分）：能区分类型，给出主要方案
- 一般（3 分）：知道冷启动概念，方案较少
- 较差（≤2 分）：理解模糊或方案不可行

## 题目 4：稀疏性问题

**问题：** 用户 - 物品评分矩阵通常非常稀疏（<1%），这会带来什么问题？有哪些解决方案？

**参考答案：**

**稀疏性带来的问题：**

1. **相似度计算不准确**：共同评分物品太少，相似度噪声大
2. **推荐覆盖度低**：很多用户/物品无法找到足够邻居
3. **长尾物品难推荐**：冷门物品几乎没有被评分
4. **计算资源浪费**：大量零值参与计算

**解决方案：**

**1. 矩阵分解（Matrix Factorization）**
```python
# 将稀疏矩阵分解为低维稠密矩阵
# R ≈ U × V^T
# U: 用户隐因子矩阵 (m × k)
# V: 物品隐因子矩阵 (n × k)
# k << m, n（降维）

# 损失函数：
min Σ(r_ui - u_u^T · v_i)^2 + λ(||U||² + ||V||²)
```

**2. 降维技术**
- SVD（奇异值分解）
- PCA（主成分分析）
- 自动编码器

**3. 图方法**
- 将用户 - 物品视为二分图
- 使用图随机游走（Random Walk）
- Graph Neural Network

**4. 数据增强**
- 隐式反馈补充（点击、浏览时长）
- 跨域数据迁移
- 合成数据生成

**5. 局部方法**
- 只考虑 K 近邻，忽略远距离关系
- 聚类后在簇内计算

**6. 深度学习**
- Neural CF 用神经网络学习非线性关系
- 自编码器填充缺失值

**工业实践：**
```
Netflix Prize 解决方案：
├── 矩阵分解（SVD++）
├── 受限玻尔兹曼机（RBM）
├── KNN 局部模型
├── 时序模型
└── 模型融合（加权平均）
```

- 良好（4 分）：能列出主要方案
- 一般（3 分）：知道矩阵分解
- 较差（≤2 分）：理解片面
