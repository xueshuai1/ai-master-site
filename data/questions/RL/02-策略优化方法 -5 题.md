# 策略优化方法面试题（5 道）

## 题目 1：基础概念
**问题：** 什么是策略优化？基于价值的方法和基于策略的方法有什么区别？

**参考答案：**
**策略优化**：直接对策略参数θ进行优化，最大化期望累积奖励 J(θ) = E_τ[R(τ)]。

**两类方法的区别：**

| 方面 | 基于价值 | 基于策略 |
|------|----------|----------|
| 优化对象 | 价值函数 Q(s,a) 或 V(s) | 策略参数θ |
| 策略获取 | 贪婪：π(s)=argmax_a Q(s,a) | 直接参数化π_θ(a|s) |
| 动作空间 | 适合离散 | 适合连续 |
| 收敛性 | 可能震荡 | 更稳定 |
| 代表性算法 | Q-Learning, DQN | REINFORCE, PPO |
| 方差 | 低 | 高（需基线减少） |

**优缺点：**
- 基于价值：样本效率高，但难以处理连续动作
- 基于策略：天然支持连续动作和随机策略，但方差大

**评分要点：** 说出优化对象区别得基本分，能列表对比得高分

## 题目 2：策略梯度定理
**问题：** 写出策略梯度定理的数学表达式，并解释为什么可以引入基线函数？

**参考答案：**
**策略梯度定理：**
```
∇_θ J(θ) = E_τ∼π_θ[Σ_t ∇_θ log π_θ(a_t|s_t) Q^π(s_t, a_t)]
```

**引入基线：**
```
∇_θ J(θ) = E_τ∼π_θ[Σ_t ∇_θ log π_θ(a_t|s_t) (Q^π(s_t, a_t) - b(s_t))]
```

**为什么可以引入基线：**
因为基线项的期望为 0：
```
E_a∼π[∇_θ log π_θ(a|s) · b(s)] 
= b(s) · E_a∼π[∇_θ log π_θ(a|s)]
= b(s) · ∇_θ E_a∼π[1]
= b(s) · ∇_θ 1
= 0
```

**基线的作用：**
- 减少梯度估计的方差
- 不引入偏差（期望仍为 0）
- 常用基线：V^π(s)、平均奖励

**评分要点：** 写出公式得基本分，证明基线期望为 0 得高分

## 题目 3：Actor-Critic
**问题：** Actor-Critic 方法中 Actor 和 Critic 分别做什么？TD 误差δ的作用是什么？

**参考答案：**
**Actor（演员）：**
- 负责选择动作：a ∼ π_θ(·|s)
- 通过策略梯度更新参数
- 目标：提高好动作的概率，降低坏动作的概率

**Critic（评论家）：**
- 评估当前策略的价值：V_φ(s)
- 通过 TD 学习更新
- 为 Actor 提供"建议"（优势估计）

**TD 误差δ的作用：**
```
δ_t = r_t + γ V(s_{t+1}) - V(s_t)
```

1. **优势估计**：δ ≈ A(s,a) = Q(s,a) - V(s)
2. **低方差信号**：比完整回报 G_t 方差更小
3. **更新方向**：
   - δ > 0：动作比预期好，增加概率
   - δ < 0：动作比预期差，降低概率

**更新规则：**
```
Actor: θ ← θ + α ∇_θ log π_θ(a_t|s_t) · δ_t
Critic: φ ← φ + β δ_t ∇_φ V_φ(s_t)
```

**评分要点：** 说出 Actor 选动作、Critic 评估得基本分，解释δ作为优势估计得高分

## 题目 4：代码实现
**问题：** 请实现 REINFORCE 算法的更新函数，并解释关键步骤。

**参考答案：**
```python
import torch
import torch.nn as nn
import torch.optim as optim

def reinforce_update(policy, optimizer, log_probs, rewards, gamma=0.99):
    """
    REINFORCE 算法更新
    
    参数:
        policy: 策略网络
        optimizer: 优化器
        log_probs: 各步的 log π(a|s)
        rewards: 各步的奖励
        gamma: 折扣因子
    """
    # 1. 计算折扣回报 G_t
    returns = []
    G = 0
    for r in reversed(rewards):
        G = r + gamma * G
        returns.insert(0, G)
    
    returns = torch.FloatTensor(returns)
    
    # 2. 标准化回报（减少方差）
    returns = (returns - returns.mean()) / (returns.std() + 1e-8)
    
    # 3. 计算策略梯度损失
    # J(θ) = E[log π(a|s) · G]
    # 梯度上升 → 最小化 -log π(a|s) · G
    policy_loss = -(torch.stack(log_probs) * returns).mean()
    
    # 4. 反向传播和更新
    optimizer.zero_grad()
    policy_loss.backward()
    optimizer.step()
    
    return policy_loss.item()

# 使用示例
# log_probs = [log_prob_0, log_prob_1, ...]
# rewards = [r_0, r_1, ...]
# loss = reinforce_update(policy, optimizer, log_probs, rewards)
```

**关键步骤解释：**
1. **折扣回报**：从后向前累加，G_t = r_t + γG_{t+1}
2. **标准化**：减去均值除以标准差，减少方差
3. **损失函数**：负的对数概率乘以回报（梯度上升）
4. **反向传播**：自动计算∇_θ log π_θ(a|s)

**评分要点：** 代码正确得基本分，能解释每步含义得高分

## 题目 5：算法对比
**问题：** 比较 REINFORCE、A2C 和 PPO 三种策略优化算法的特点和适用场景。

**参考答案：**

| 特性 | REINFORCE | A2C | PPO |
|------|-----------|-----|-----|
| **类型** | 纯策略梯度 | Actor-Critic | Actor-Critic |
| **更新方式** | 蒙特卡洛（完整 episode） | 时序差分（在线） | 多 epoch 小批量 |
| **方差** | 高 | 中 | 低 |
| **样本效率** | 低 | 中 | 高 |
| **稳定性** | 较差 | 较好 | 很好 |
| **关键技巧** | 基线 | TD 误差 | 截断 + 价值 clip |
| **超参数敏感** | 是 | 中 | 否 |

**REINFORCE：**
- 优点：简单，无偏
- 缺点：方差大，需要完整 episode
- 适用：简单任务，教学演示

**A2C（同步优势 Actor-Critic）：**
- 优点：低方差，在线学习
- 缺点：步长敏感，可能不稳定
- 适用：中等复杂度任务

**PPO（近端策略优化）：**
- 优点：稳定，样本高效，超参数鲁棒
- 缺点：实现稍复杂
- 适用：大多数现代 RL 应用（首选）

**选择建议：**
- 入门/教学：REINFORCE
- 快速原型：A2C
- 生产环境：PPO

**评分要点：** 说出三种算法类型得基本分，详细对比表格得高分
