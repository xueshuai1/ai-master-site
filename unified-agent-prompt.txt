**🔴 优先级判断框架（按价值排序，不是按难易）：**

| 优先级 | 场景 | 判断标准 |
|--------|------|---------|

**🔴 新闻铁律（重要！）：**

- ✅ 如果写了新博客/知识库文章，可以在 news.ts 中用**外部视角**写一条（如："Anthropic 发布 Claude Opus 4.7" 而不是 "本站发布 Claude Opus 4.7 解读"）
| 🟢 P3 | 内容优化 / 体验提升 | 页面排版优化、交互改进 |

**新闻是最低优先级的事。知识库和工具集的持续进化才是核心价值。**

**🔴 博客与知识库的定位差异（重要！）：**

| 维度 | 知识库 | 博客 |

## 📝 老文章更新机制

**知识库文章不仅新增，还要定期更新已有内容。**

**触发条件：**
- 内容研究报告中注明"建议更新文章 xxx"（如某主题有新进展）
- 某篇文章内容过时（如工具版本更新、方法被替代）
- 热点新闻涉及已有知识库文章（如 LoRA 有重大更新）

**更新方式：**
- 不删除原有内容，在原有章节末尾追加"更新于 YYYY-MM-DD：..."
- 新增"最新动态"章节，记录该主题的最新进展
- 更新文章的 tags、summary（如有变化）
- 不修改文章 ID 和基础结构

**更新频率：**
- 热门主题（LLM、Agent、RAG 等）：每月检查一次
- 成熟主题（数学基础、经典 ML 算法）：每季度检查一次

**注意：**
- 更新时保持文章原有的风格和结构
- 如果有事实错误（如版本号、API 变更），直接修正
- 每次更新要在 commit message 注明更新了哪些文章

### 5. ⭐ QA 自测（必须执行）
用 browser 打开网站（本地或 Vercel），**按以下清单逐项检查**：

**🔴 移动端必检（390px 视口 — 每次改动 UI 都必须做！）**

**🖥️ PC 端必检（1440px 视口）**
- [ ] 布局正常，内容不溢出
- [ ] 导航链接能跳转

**⚡ 交互必检**
- [ ] **快速点击不崩溃**：分类筛选、排序按钮等快速连续点击，看是否出现重复数据或失效

**🔴 布局一致性检查**
- [ ] `grep -n "max-w-" <相关文件>` 检查各区域宽度是否统一

**⚠️ 经验教训（2026-04-20 记录）：**

2. router.replace 竞态会导致列表重复 — URL 同步必须加防抖
3. 统计数不一致是因为不同页面用了不同逻辑 — 必须提取共用函数
