## 执行流程

### 1. 状态检查
cd /Users/xueshuai/.openclaw/workspace/ai-master-site
git pull origin main

### 2. 全面评估当前网站状况（这是最重要的步骤！）
不要只是"读报告"，你要像一个产品经理一样全面审视：
- **内容丰富度**：知识库有多少条？每条内容有多长？是不是只有几行字？有没有代码示例？有没有图表？跟维基百科比差多少？
- **用户体验**：打开首页会不会觉得空洞？页面有没有吸引力？
- **功能完整性**：有没有半成品？有没有死链？有没有点击没反应的按钮？
- **视觉设计**：是不是太单调？纯文字太多？有没有图标、插图、动画？
- **技术债**：有没有 build warning？有没有 console error？有没有性能问题？

### 3. 自主决策（只做一件事）
根据评估结果，选一个对网站价值最大的改动来做。**优先级你自己判断，不要等指令。**

**参考方向（但不限于此）：**
- 知识库内容太少 → 大幅增加文章数量，每条内容丰富详实
- 内容太浅 → 加入代码示例、架构图、对比表格、Mermaid 流程图
- 用户体验差 → 增加互动功能、动画、搜索优化
- 页面空洞 → 填充真实有价值的内容
- 有 bug → 修复
- 新功能 → 你觉得网站需要什么就加什么

**知识库内容标准（重点！）：**
- 每条知识必须内容丰富、有深度，不能只有几行字
- 必须包含：摘要、分章节正文、代码示例（带语法高亮）、架构图/流程图/对比表格
- 来源必须是国外权威网站（OpenAI、DeepMind、Anthropic、MIT Tech Review、TechCrunch、arXiv 等）
- 像维基百科条目一样详实，不是简单的摘要
- 每次运行至少要新增或大幅改进 1-2 条知识条目

### 4. ⭐ 本地验证（写完代码后必须执行）
运行：`npm run build`
- 如果 build 失败 → 修复到 build 通过为止
- 如果 build 通过 → 进入下一步

### 5. ⭐ QA 自测（必须执行）
用 browser 工具打开 https://ai-interview-questions-eight.vercel.app
- 打开你刚改动的页面，截图确认
- 测试改动相关的交互功能
- 如果有问题 → 回去修，修完再回到步骤 4
- 确认无问题 → 进入下一步
