### 1. Mermaid 图表配色和类型（2026-05-08 更新）

**🔴 样式语法铁律（2026-05-08 infra-002 事故新增）：**
- ❌ **禁止使用 `class X style fill:#颜色` 语法** — 这是非法语法！mermaid 中 `class` 指令不能和 `style` 混用，会导致整个图表渲染失败（💥 崩溃图标）
- ✅ **使用 `style X fill:#颜色,color:#fff`** — 直接给节点着色，简洁有效
- ✅ **或使用 `classDef` + `class`** — 先定义样式类，再批量应用
- ❌ **禁止使用浅色**（400-500 色阶）和 Material Design 浅色：`#ef4444` `#f59e0b` `#10b981` `#6366f1` `#8b5cf6` `#ec4899` `#f3e5f5` `#e1f5fe` `#c8e6c9` `#fff3e0` `#8AB4F8` `#E8EAED` `#F28B82`
- ✅ **必须使用深色**（800-900 色阶）+ 白字 `color:#fff`：`#991b1b` `#92400e` `#064e3b` `#3730a3` `#581c87` `#1e3a5f`
- 完整配色表：`docs/MERMAID-COLORS.md`

**✅ 正确示例 1（style 直接着色）：**
```
graph TB
    A[节点1] --> B[节点2]
    style A fill:#991b1b,color:#fff
    style B fill:#1e3a5f,color:#fff
```

**✅ 正确示例 2（classDef 批量着色）：**
```
graph TB
    A[节点1] --> B[节点2]
    classDef important fill:#991b1b,stroke:#b91c1c,color:#fff;
    class A important;
```

**❌ 错误示例（全部禁止！）：**
```
graph TB
    A[节点1] --> B[节点2]
    class A style fill:#991b1b  ← 禁止！非法语法，渲染崩溃（infra-002 事故）
```

**🔴 连线语法铁律（2026-04-29 blog-084 事故新增）：**
- ❌ **禁止使用 `.- >`（中间带空格）** — 会导致连线断裂、节点渲染为空白块
- ✅ **连线箭头必须是 `->` 连续字符**，如 `A --> B`、`A -.标注.-> B`
- ❌ **禁止** `.- >`、`. - >`、`. -  >` 等任何带空格变体
- ✅ **虚线标注箭头格式**：`A -.text.-> B`（注意 `.` 在 text 前后都要有）

**🔴 % 符号铁律（2026-04-29 blog-084 控制台报错事故新增）：**
- ❌ **禁止在 mermaid 块中使用 `%` 字符** — mermaid 中 `%` 是注释标记，会导致整行解析错误
- ✅ **必须使用全角 `％`** 代替半角 `%`，如 `15-25％` 而非 `15-25%`
- ❌ **`\%` 转义无效** — JS 模板字面量不识别 `\%`，会静默丢掉反斜杠，mermaid 仍然看到 `%`

**🔴%% 注释符铁律（2026-05-14 ai-law-001 事故新增）：**
- ❌ **禁止在 mermaid `%%{init:...}%%` 注释配置中使用全角 `％％`** — 必须用半角 `%%`
- ✅ `%%{init: {'theme': 'neutral', ...}}%%` 中的 `%%` 必须是半角 ASCII 百分号
- ✅ 只有图表内容中的百分号才用全角 `％`（如 `95％`），注释配置的 `%%` 必须半角

**✅ 合法的 Mermaid 图表类型（只能用以下类型）：**
- `graph TD` / `graph LR` / `graph BT` / `graph RL`（流程图/架构图）— **最推荐**
- `sequenceDiagram`（时序图）
- `pie`（饼图）
- `stateDiagram-v2`（状态图）
- `classDiagram`（类图）
- `erDiagram`（实体关系图）
- `journey`（用户旅程图）
- `quadrantChart`（象限图）
- `mindmap`（思维导图）

**⚠️ 慎用类型：**
- `gantt`（甘特图）— 中文标签过长会导致文字重叠、日期截断，**不推荐用于中文内容**。如需展示时间线/路线图，改用 `graph LR` + subgraph 分组

**❌ 非法的 Mermaid 图表类型（禁止使用）：**
- `tree-beta` — 不是合法类型，会渲染为空白块
- `flowchart` 后必须跟方向：`flowchart TD`，不能单独用 `flowchart`
- 任何不在上述合法列表中的类型

### 2. 代码块格式（最高频出错！）
- 🔴 **代码块是可选的，但 Mermaid 图表是必须的。** 文章可以没有代码，但不能没有图表。
- ❌ **禁止**在 body 中写 `\`\`\`lang ... \`\`\` 代码块 — MarkdownBody 不会渲染语法高亮
- ✅ **必须**把代码放在 `code: [{ lang, code, title }]` 数组中
- body 中用**描述性文字**指向代码，如「（见下方代码：LessHumanAgent 完整实现）」
- **每个 code 块必须有 `title` 字段**，与 body 中的引用文字对应

**正确示例（整个 section）：**
```

  body: `下面我们用 Python 实现。

### 实现 1：核心 Agent

（见下方代码：LessHumanAgent 完整实现）

### 实现 2：约束验证

（见下方代码：约束验证器实现）`,
  code: [
    {
      lang: "python",
      title: "LessHumanAgent 完整实现",
      code: `import json
import time
...
# 完整代码放这里`,
    },
    {
      lang: "python",
      title: "约束验证器实现",
      code: `from enum import Enum
...
# 完整代码放这里`,
    },
  ],
}
```

**错误示例（不要这样写！）：**
```
// ❌ 错误1：代码嵌在 body 里
body: `文字...
\`\`\`python
import json
\`\`\`
...文字`,

// ❌ 错误2：code 块没有 title
code: [{
  lang: "python",
  code: `import json...`,
}],

// ❌ 错误3：body 用「见代码块1」这种无意义引用
body: `...（见代码块 1）...`,
```
