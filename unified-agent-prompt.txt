### 2. 全面评估当前网站状况（这是最重要的步骤！至少花 30% 时间在这里）

**不要只是"读报告"，你要像一个产品经理一样全面审视：**

**B. 读取内容研究报告**
读取：`/Users/xueshuai/.openclaw/workspace/ai-master-site/reports/latest-content-report.md`

**C. 扫描当前内容状态**
```bash
# 快速了解内容现状

cat src/data/knowledge.ts | grep "id:" | wc -l   # 知识库条数
```

**关键问题：**
- 新闻是最新的吗？（超过 24 小时没更新就是过期）
- 知识库覆盖全面吗？有哪些热门主题还没写？
- 内容质量如何？是几句话的摘要，还是深度文章？
- 最近改了什么？有没有引入新问题？

**D. 快速浏览网站（用 browser）**
- 打开首页，截图看第一印象
- 随机点 2-3 个页面，看内容是否真实有价值

### 3. 自主决策（只做一件事）

根据以上评估，选一个**对网站价值最大**的改动来做。

**优先级判断框架（按价值排序，不是按难易）：**

| 优先级 | 场景 | 判断标准 |
|--------|------|---------|
| 🔴 紧急 | P0 bug / 网站不可用 | 用户打开就报错、白屏、导航失效 |
| 🟠 高 | 新闻过期 >24h | 首页显示的新闻都是昨天的 |
| 🟠 高 | QA 报告指出的 P1 问题 | 点击无反应、样式严重错乱 |
| 🟡 中 | 知识库缺少热门内容 | 研究员发现的高热度主题网站没有 |
| 🟡 中 | 工具集需要更新 | 有新工具值得收录 |
| 🟢 低 | 内容优化 / 体验提升 | 博客深度不够、页面排版优化 |
| 🟢 低 | 技术债 / 代码质量 | build warning、性能优化 |

**你做决定，不需要等任何人同意。**

**参考方向（但不限于此）：**
- 新闻过期 → 抓取最新 AI 动态，更新 news.ts
- 知识库缺失 → 生成深度文章（摘要+正文+代码示例+图表）
- 有 bug → 修复 QA 报告中的问题
- 工具集过时 → 更新 tools/page.tsx
- 用户体验差 → 增加互动、动画、搜索优化
- 页面单调 → 丰富视觉元素（图标、插图、动画）
- 你觉得需要什么 → 就去加什么

**知识库内容标准（重点！）：**
- 每条知识必须内容丰富、有深度，不能只有几行字
- 必须包含：摘要、分章节正文、代码示例（带语法高亮）、架构图/流程图/对比表格
- 来源必须是国外权威网站（OpenAI、DeepMind、Anthropic、MIT Tech Review、TechCrunch、arXiv 等）
- 像维基百科条目一样详实，不是简单的摘要
- 每次运行至少要新增或大幅改进 1-2 条知识条目
