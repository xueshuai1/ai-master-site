## 题目描述
分布式强化学习通过并行化加速训练，请说明分布式 RL 的主要架构和设计考虑。

**标签：** 分布式 RL, 并行训练，IMPALA, 架构设计

## 参考答案

**并行化维度：**

1. **数据并行：**
   - 多个 worker 收集数据
   - 梯度聚合更新
   - 例：A3C, A2C

2. **模型并行：**
   - 大模型分割到多个设备
   - 适用于超大网络

3. **环境并行：**
   - 多个环境实例并行运行
   - 最常用，易于实现

**主要架构：**

1. **同步架构（A2C）：**
   - 所有 worker 同步更新
   - 简单但等待慢 worker

2. **异步架构（A3C）：**
   - Worker 独立更新
   - 无需等待，但梯度可能过时

3. **Actor-Learner 分离（IMPALA）：**
   - Actor 只收集数据
   - Learner 只训练
   - 用 V-trace 校正策略差异

4. **分布式 replay buffer：**
   - 共享经验回放池
   - 例：Distributed DQN, R2D2

**设计考虑：**

1. **通信开销：**
   - 梯度/参数传输成本
   - 压缩、异步通信

2. **数据新鲜度：**
   - 过时梯度问题
   - 重要性采样校正

3. **负载均衡：**
   - 不同环境速度不同
   - 动态调度

4. **容错：**
   - Worker 失败处理
   - 检查点恢复

**代表系统：**
- IMPALA: 高效异步
- Seed RL: 集中式 inference
- CleanRL: 简洁实现

## 考察重点
- 分布式系统设计
- 并行化策略
- 工程实践能力

## 延伸追问
### 追问 1: V-trace 校正
**问题：** IMPALA 中 V-trace 的作用是什么？
**答案：** 校正行为策略与目标策略的差异，允许 off-policy 学习。

### 追问 2: 扩展性
**问题：** 分布式 RL 的扩展性如何？
**答案：** 可以扩展到数千 CPU 核，但通信可能成为瓶颈。

## 深入理解
分布式 RL 是实现大规模训练的基础设施，对于复杂任务必不可少。
