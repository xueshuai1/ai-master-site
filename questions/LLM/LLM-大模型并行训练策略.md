## 题目描述
请解释大模型训练中的并行策略，包括数据并行、张量并行、流水线并行的原理和组合方式。

**来源：** LLM 面试题整理
**标签：** 并行训练，数据并行，模型并行，流水线

## 参考答案

**并行训练需求：**
- 模型太大，单卡放不下
- 数据太多，单机训练太慢
- 需要多机多卡协同训练

**三种并行策略：**

**1. 数据并行（Data Parallelism）：**
```
GPU0: 模型副本 + 数据批次 0 → 梯度 0
GPU1: 模型副本 + 数据批次 1 → 梯度 1
GPU2: 模型副本 + 数据批次 2 → 梯度 2
       ↓ All-Reduce 聚合梯度
所有 GPU: 更新相同模型
```
- **原理：** 每卡保存完整模型，处理不同数据
- **通信：** 每步 All-Reduce 梯度
- **适用：** 模型能放入单卡
- **扩展性：** 受限于单卡显存

**2. 张量并行（Tensor Parallelism）：**
```python
# 将矩阵乘法拆分到多卡
# Y = XA + B

# 卡 0: Y0 = XA0 + B0
# 卡 1: Y1 = XA1 + B1
# 输出：Y = concat(Y0, Y1)

# 典型切分位置：
# - Attention 的 QKV 投影
# - FFN 的第一层和第二层
```
- **原理：** 将单层内的矩阵拆分到多卡
- **通信：** 层内频繁通信（每算子）
- **适用：** 超大模型单层
- **典型：** Megatron-LM

**3. 流水线并行（Pipeline Parallelism）：**
```
GPU0: Layer 1-8   → 微批次 0,1,2,3
GPU1: Layer 9-16  → 微批次 0,1,2,3
GPU2: Layer 17-24 → 微批次 0,1,2,3
GPU3: Layer 25-32 → 微批次 0,1,2,3
```
- **原理：** 将模型层切分到不同卡
- **通信：** 层间传递激活和梯度
- **挑战：** 流水线气泡（GPU 空闲）
- **优化：** GPipe、1F1B、Interleaved

**3D 并行（组合策略）：**
```
总 GPU 数 = 数据并行度 × 张量并行度 × 流水线并行度

示例（训练 175B 模型）：
- 数据并行：8
- 张量并行：8
- 流水线并行：16
- 总 GPU：8 × 8 × 16 = 1024
```

**ZeRO 优化（DeepSpeed）：**
```
ZeRO-1: 优化器状态分片
ZeRO-2: 优化器 + 梯度分片
ZeRO-3: 优化器 + 梯度 + 参数分片
```
- 进一步减少显存占用
- 支持更大模型

**通信优化：**
- 梯度累积：减少通信频率
- 混合精度：减少通信量
- 重叠计算和通信：隐藏延迟

**实践建议：**
```python
# 模型 < 单卡显存：纯数据并行
# 模型 > 单卡显存：张量并行 + 数据并行
# 模型 >> 单卡显存：3D 并行 + ZeRO
```
