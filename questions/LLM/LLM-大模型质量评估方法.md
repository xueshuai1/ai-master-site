## 题目描述
如何全面评估大模型的质量？请从多个维度说明评估指标、基准测试和评估方法。

**来源：** LLM 面试题整理
**标签：** 评估，基准测试，指标，排行榜

## 参考答案

**评估维度：**

**1. 语言能力：**
- **MMLU：** 57 个学科的多任务理解（0-shot/5-shot）
- **BIG-Bench：** 200+ 多样化任务
- **SuperGLUE：** 语言理解基准

**2. 推理能力：**
- **GSM8K：** 小学数学应用题
- **MATH：** 高中至大学数学
- **HumanEval：** 代码生成

**3. 知识能力：**
- **TruthfulQA：** 事实性和抗误导性
- **NaturalQuestions：** 开放域问答

**4. 长文本处理：**
- **Needle In A Haystack：** 长文档信息检索
- **LongBench：** 长文本理解基准

**5. 多语言能力：**
- **FloRes：** 翻译质量
- **XCOPA：** 多语言推理

**评估指标：**

| 任务类型 | 指标 | 说明 |
|----------|------|------|
| 生成任务 | BLEU/ROUGE | n-gram 重叠率 |
| 问答任务 | Exact Match/F1 | 答案匹配度 |
| 分类任务 | Accuracy/F1 | 分类准确率 |
| 排序任务 | NDCG/MRR | 排序质量 |
| 代码任务 | Pass@k | 通过率 |

**评估方法：**

**1. 自动评估：**
```python
# 使用标准基准测试
from lm_eval import evaluator
results = evaluator.simple_evaluate(
    model="llama-7b",
    tasks=["mmlu", "gsm8k", "humaneval"],
    num_fewshot=5
)
```

**2. 人工评估：**
- 盲测对比（A/B Test）
- 多维度打分（有用性、安全性、流畅性）
- 专家评估（特定领域）

**3. LLM-as-a-Judge：**
```
用更强模型评估目标模型输出
评分维度：准确性、完整性、安全性
```

**4. 红队测试：**
- 对抗性提示测试
- 安全性边界探索
- 偏见和毒性检测

**评估挑战：**
- **数据污染：** 测试集可能在训练数据中
- **指标局限：** 自动指标与人类偏好不一致
- **成本高昂：** 全面评估需要大量计算
- **动态基准：** 模型快速迭代，基准过时

**最佳实践：**
- 多基准综合评估，避免单一指标
- 结合自动和人工评估
- 报告评估设置（shot、temperature 等）
- 开源评估代码便于复现

**应用：**
- 模型选型和对比
- 训练过程监控
- 发布前质量把关
