## 题目描述
请解释指令微调（Instruction Tuning）的目的和方法，以及主流的对齐技术（RLHF、DPO 等）。

**标签：** 指令微调，对齐，SFT，DPO

## 参考答案

**指令微调目的：**
让模型学会遵循人类指令，而非仅仅续写文本。

**指令微调流程：**
```
预训练模型 → 指令数据 SFT → 对齐训练 → 可用模型
```

**指令数据构建：**
```python
# 数据格式
{
    "instruction": "请总结以下文章",
    "input": "文章正文...",
    "output": "总结内容..."
}

# 数据来源
- 人工编写（高质量，成本高）
- 自指令（Self-Instruct）：用模型生成
- 开源数据集（Alpaca、Dolly 等）
- 蒸馏自更强模型
```

**对齐技术对比：**

**1. RLHF（Reinforcement Learning from Human Feedback）：**
```
阶段 1: SFT（监督微调）
阶段 2: 训练奖励模型（人类偏好排序）
阶段 3: PPO 优化（最大化奖励 + KL 惩罚）

优点：效果好，经过验证
缺点：复杂，不稳定，需要奖励模型
```

**2. DPO（Direct Preference Optimization）：**
```python
# 直接从偏好数据优化，无需奖励模型
loss = -log(σ(β·log(π(y_w|x)/π_ref(y_w|x)) - 
              β·log(π(y_l|x)/π_ref(y_l|x))))

# y_w: 偏好的回答，y_l: 不偏好的回答
# π_ref: 参考模型（通常是 SFT 模型）
```
- 优点：简单稳定，无需奖励模型
- 缺点：理论假设较强

**3. IPO（Identity Preference Optimization）：**
```
DPO 的变体，避免过拟合偏好数据
```

**4. KTO（Kahneman-Tversky Optimization）：**
```
基于前景理论，无需成对偏好数据
只需标注"好"或"坏"
```

**技术对比：**

| 方法 | 复杂度 | 数据需求 | 效果 | 稳定性 |
|------|--------|---------|------|--------|
| RLHF | 高 | 偏好排序 | 最佳 | 中 |
| DPO | 中 | 偏好排序 | 好 | 高 |
| KTO | 低 | 单点标注 | 中 | 高 |

**实践建议：**
```python
# 资源充足：RLHF（PPO）
# 追求稳定：DPO
# 数据有限：KTO
# 快速迭代：DPO + 高质量 SFT

# DPO 训练配置
{
    "beta": 0.1,  # 控制偏离参考模型的程度
    "loss_type": "sigmoid",
    "reference_free": False  # 使用参考模型
}
```

**评估对齐效果：**
- 胜率评估（Elo 评分）
- 安全性测试（红队）
- 有用性评分
- 人工评估
